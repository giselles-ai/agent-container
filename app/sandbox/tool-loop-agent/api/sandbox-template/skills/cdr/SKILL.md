# CDR作成スキル

CDR (Corporate Decision Record) を作成するスキルです。ユーザーから必要な情報を収集し、適切なフォーマットでCDRディレクトリとファイルを生成します。

## 手順

### 1. 情報収集

以下の情報をユーザーから収集してください：

**必須項目：**
- タイトル（内容を端的に説明したもの、句点・空白なしの日本語で最大40文字程度）
- 承認内容（何を承認してほしいのか）
- 起案背景（どうして必要なのか）
- 部門名

**状況に応じて収集：**
- 決裁区分ID（該当する場合）
- 項目（「収益」または「費用」）
- 金額(税抜)
- 支払方法（'経費精算'、'クレジットカード'、'請求書'、'その他' のいずれか）
- 会計科目
- 契約先
- 契約コード
- 購入・契約開始年月（yyyy/mm形式）
- 契約終了年月（yyyy/mm形式）
- 発注担当者（GitHubユーザー名）
- 期待効果
- 代替手段との比較
- リスク要因とその対応
- 金額内訳
- 予実への影響
- 事後モニタリング
- 関連Issues/PRs
- 添付書類

### 2. 決裁区分IDの特定

決裁区分IDを特定するには、以下のコマンドで最新情報を取得：
```bash
gh api repos/r06-cdr/cdr-assets/contents/decision_categories.tsv --jq '.content' | base64 -d
```

主な決裁区分：
- 経費支出（100万円以上）: ID 75（全取締役承認）
- 経費支出（100万円未満、開発サービス/ツール関連）: ID 77
- 経費支出（100万円未満、社内IT関連）: ID 78
- 経費支出（100万円未満、その他）: ID 79
- 深夜残業申請、休日出勤申請: ID 87
- 副業申請: ID 88
- 顧客への最終成果物: ID 76
- リーガルチェック: ID 83
- 押印関連: ID 84, 85

### 3. ディレクトリとファイルを作成

以下の構造でディレクトリとREADME.mdを作成します：

```
cdr/
└── <タイトル>/
    └── README.md
```

**ディレクトリ名の規則：**
- タイトルをそのままディレクトリ名として使用
- タイトルはスペースを含まない（アンダースコアやハイフンで代替）

**注意：**
- CDR IDの採番はCIで自動的に行われるため、ディレクトリ名にIDを含めない
- cdr_metadata.jsoncはCIで自動生成されるため、作成しない

### 4. README.mdのテンプレート

```markdown
---
# 必要に応じて内容を記述、使用しないものについては空欄
# 各項目の詳細は https://github.com/r06-cdr/cdr-assets/blob/main/README.md#Frontmatter を参照

決裁区分ID:
部門:
項目:
金額(税抜):
支払方法:
会計科目:
契約先:
契約コード:
購入・契約開始年月:
契約終了年月:
発注担当者:
cdr_meta_version: 3
---

<!-- テンプレートの見出し項目は削除せず、入力しない場合はその理由を記述 -->

# タイトル

# 承認内容（何を承認してほしいのか）

# 起案背景（どうして必要なのか）

# 期待効果（どんな効果が期待できるのか）

# 代替手段との比較（なぜそれを選んだのか）

# リスク要因とその対応（どんなリスクがあり、どう対応するのか）

# 金額内訳（具体的にどんな収益/費用が発生するのか）

# 予実への影響（予算内/予算外であるのか）

# 事後モニタリング（期限等の管理が必要になるか）

# 関連Issues/PRs

# 添付書類

```

### 5. 作成後の案内

CDR作成後、以下を案内してください：

1. 作成されたファイルの確認
2. 必要に応じて内容を編集
3. ブランチを作成してコミット
4. Pull Requestを作成し、適切な承認者をレビュアーに設定

## 決裁区分の詳細（参考）

| 分類 | 説明 |
|------|------|
| A.CDR外承認 | freee、KING OF TIME、クラウドサインなど外部システムで承認 |
| B.全取締役承認 | 重要な経営判断、100万円以上の支出など |
| A.B.以外 | 上記以外の一般的な承認（このリポジトリで処理） |

## 注意事項

- Frontmatterの `cdr_meta_version` は必ず `3` を設定
- 入力しない項目は空欄のままにする（削除しない）
- テンプレートの見出し項目は削除せず、入力しない場合はその理由を記述
- 添付ファイルがある場合はREADME.mdと同じディレクトリに配置
